# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 仓库用途

这是一个周报仓库，用于根据每周的实验记录和工作事项自动生成周报。

## 目录结构

- `template.md` — 周报的 Markdown 模板
- `config.json`（可选）— 仓库级配置文件，支持以下字段：
  - `output_dir`：周报输出目录，可以是相对路径（相对于仓库根目录）或绝对路径。若不设置，默认保存到对应日期文件夹下
- 每周一个日期文件夹（如 `2026.3.1/`），文件夹内包含：
  - **link.txt** — 每行一条，格式为 `文档标题 : 链接URL`，对应飞书实验报告的在线链接
  - **extra_terms.txt**（可选）— 没有实验报告对应的其他工作事项，每行一条
  - **PDF 文件**（可选）— 如果飞书 MCP 不可用时的备用方式，文件名格式为 `[日期] 标题.pdf`

## 周报生成规则

当用户要求生成某一周的周报时，按以下步骤操作：

1. **读取对应日期文件夹下的所有材料**：link.txt（获取实验报告链接）、extra_terms.txt（获取额外事项），以及 PDF 文件（若存在）
   - **优先通过飞书 MCP 读取文档内容**：读取 `link.txt` 中的每条链接，使用飞书 MCP 工具直接获取对应飞书文档的内容，无需依赖 PDF 文件
   - **降级处理**：若飞书 MCP 不可用，则回退到读取文件夹中的 PDF 文件
   - **增量处理**：实验报告可能是增量式的——本周的某个文档可能与之前某周同名但内容有更新。此时需要同时读取之前同名的文档，对比差异，周报中只体现本周新增或变化的部分，不重复描述之前已有的内容
2. **使用 `template.md` 作为输出格式**
3. **填写各部分内容**：
   - **一周工作总结**：用 `-` bullet list 格式概括本周主要工作方向和研究内容
   - **完成工作事项**：列出已完成的工作，每项一行。注意以下几点：
     - **动词准确**：根据实际工作性质选择动词——仅调用/评估现有工具用"测试"，自己搭建环境后测试用"搭建并测试"，已完成功能开发用"实现了"等
     - **问题列举**：多个问题或缺点用引号逐条列举，格式为 `"问题1"，"问题2"和"问题3"等问题`
     - **链接附加**：如果多条事项共属同一个实验报告，在这批事项末尾另起一行写 `以上结果可见 [[实验报告]](链接)`；如果某条事项单独对应一个报告，则在该条末尾直接附链接
   - **待完成工作事项**：不要机械搬运 PDF 中的 todo，只在用户明确要求时才列出
4. **输出为 Markdown 文件**，命名为 `周报_日期.md`，保存位置按以下优先级决定：
   - 若 `config.json` 存在且配置了 `output_dir`，则保存到该目录（目录不存在时自动创建）
   - 否则保存到对应日期文件夹下
